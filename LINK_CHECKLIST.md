# Public Link Checklist

Devpost judges need links that work outside this ChatGPT workspace. Local `sandbox:` links are not public and should not be submitted.

## Required Public Links

| Item | Target | Status |
| --- | --- | --- |
| Live demo | GitHub Pages, Vercel, or Netlify | TODO |
| Code repository | Public GitHub repo or private repo shared with judges | TODO |
| YouTube demo | Public YouTube video under 3 minutes | TODO |
| Codex session | `/feedback` Session ID | TODO |

## Recommended GitHub Pages Setup

1. Create a GitHub repository named `veyanet-decision-gate`.
2. Upload all files in this folder to the repository root.
3. Open repository settings.
4. Go to **Pages**.
5. Set source to **Deploy from a branch**.
6. Select `main` branch and `/root`.
7. Save.
8. Wait for the Pages URL to appear.

Expected live URL:

```text
https://YOUR_GITHUB_USERNAME.github.io/veyanet-decision-gate/
```

## If The Repository Must Stay Private

Share the private repository with:

```text
testing@devpost.com
build-week-event@openai.com
```

For the fastest low-friction judging path, a public repo plus GitHub Pages is usually simpler.
